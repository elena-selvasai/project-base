# /spec 명령어

$ARGUMENTS 기능에 대한 SPEC.md 명세서를 작성합니다.

## 개요

기능 설명을 입력받아 specs/[feature-name]/SPEC.md 파일을 생성합니다.

## 실행 단계

1. 기능 이름 생성: 입력된 설명에서 2-4 단어의 짧은 이름 추출
2. 디렉토리 생성: specs/[feature-name]/ 디렉토리 생성
3. SPEC.md 작성

## SPEC.md 필수 섹션

### 1. 개요 (Overview)
- 기능의 목적과 필요성
- 범위 (포함/제외 사항)

### 2. 사용자 스토리 (User Stories)
- As a [사용자 유형]
- I want to [행동]
- So that [이점]

### 3. 요구사항 (Requirements)
- 기능적 요구사항 (FR): 구체적이고 측정 가능하게
- 비기능적 요구사항 (NFR): 성능, 보안, 사용성

### 4. 인수 조건 (Acceptance Criteria)
- 체크리스트 형태로 테스트 가능한 조건들

### 5. 데이터 모델 (해당시)
- 엔티티, 관계, 스키마 변경

### 6. API/인터페이스 (해당시)
- 엔드포인트 정의
- 요청/응답 예시
- 에러 케이스

### 7. 엣지 케이스
- 예외 상황과 처리 방법

### 8. 보안 고려사항
- 인증/인가, 입력 검증 등

## 작성 원칙

해야 할 것:
- WHAT (무엇을)과 WHY (왜)에 집중
- 구체적이고 측정 가능하게 작성
- 사용자 관점에서 기술
- 검증 가능한 인수 조건 정의

하지 말아야 할 것:
- HOW (어떻게) 구현할지 언급 금지
- 기술 스택, API, 코드 구조 언급 금지
- 모호한 표현 피하기

## 불명확한 부분 처리

- 합리적인 추측으로 기본값 설정
- 가정(Assumptions) 섹션에 기록
- 정말 중요한 결정만 NEEDS CLARIFICATION 마커 사용 (최대 3개)

## 완료 후

1. 생성된 SPEC.md 경로 보고
2. 다음 단계 안내: /plan specs/[feature-name]/SPEC.md
