# /spec 명령어

$ARGUMENTS 기능에 대한 SPEC.md 명세서를 작성해주세요.

## 지침

1. **파일 위치**: `specs/[feature-name]/SPEC.md`에 생성
2. **참조 파일**: 
   - `constitution.md` - 프로젝트 규칙 준수
   - 기존 SPEC 파일들 참조하여 일관된 스타일 유지

## SPEC.md 필수 섹션

다음 섹션들을 포함해서 작성하세요:

### 1. 개요 (Overview)
- 기능의 목적과 필요성
- 범위 (포함/제외 사항)

### 2. 사용자 스토리 (User Stories)
```
**As a** [사용자 유형]
**I want to** [행동]
**So that** [이점]
```

### 3. 요구사항 (Requirements)
- **기능적 요구사항 (FR)**: 구체적이고 측정 가능하게
- **비기능적 요구사항 (NFR)**: 성능, 보안, 사용성

### 4. 인수 조건 (Acceptance Criteria)
- 체크리스트 형태로 테스트 가능한 조건들
- `- [ ]` 형식 사용

### 5. 데이터 모델 (해당시)
- 엔티티, 관계, 스키마 변경

### 6. API/인터페이스 (해당시)
- 엔드포인트 정의
- 요청/응답 예시
- 에러 케이스

### 7. 엣지 케이스
- 예외 상황과 처리 방법

### 8. 보안 고려사항
- 인증/인가, 입력 검증 등

## 작성 원칙

- 구체적이고 측정 가능하게 작성
- 사용자 관점에서 기술
- 모호한 표현 피하기 (예: "빠르게" → "200ms 이내")
- 검증 가능한 인수 조건 정의
