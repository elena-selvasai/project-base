# /checklist 명령어

$ARGUMENTS 도메인에 대한 요구사항 품질 체크리스트를 생성합니다.

## 핵심 개념: "영어로 작성된 요구사항의 단위 테스트"

체크리스트는 구현이 아닌 요구사항 자체의 품질을 검증합니다.

### 하지 말아야 할 것 (구현 테스트)
- "버튼이 올바르게 클릭되는지 확인"
- "에러 핸들링이 작동하는지 테스트"
- "API가 200을 반환하는지 확인"

### 해야 할 것 (요구사항 품질 검증)
- "모든 카드 유형에 대해 시각적 계층 구조 요구사항이 정의되어 있는가?"
- "'prominent display'가 구체적인 크기/위치로 수치화되어 있는가?"
- "모든 대화형 요소에 대해 hover 상태 요구사항이 일관되게 정의되어 있는가?"
- "키보드 네비게이션에 대한 접근성 요구사항이 정의되어 있는가?"

## 실행 단계

1. **컨텍스트 로드**
   - spec.md 읽기 (기능 요구사항과 범위)
   - plan.md 읽기 (기술 세부사항, 의존성) - 있는 경우
   - tasks.md 읽기 (구현 작업) - 있는 경우

2. **사용자 의도 파악**
   - 체크리스트 테마 도출 (예: security, ux, api, performance)
   - 사용자가 명시한 필수 항목 통합
   - spec/plan/tasks에서 누락된 컨텍스트 추론

3. **체크리스트 생성**
   - specs/[feature-name]/checklists/ 디렉토리 생성 (없는 경우)
   - 도메인 기반 파일명 생성 (예: ux.md, api.md, security.md)
   - CHK001부터 순차 번호 부여

## 체크리스트 형식

```markdown
# [도메인] 요구사항 품질 체크리스트

**목적**: [체크리스트 목적]
**생성일**: [날짜]
**기능**: [spec.md 링크]

## 요구사항 완전성 (Completeness)
- [ ] CHK001 - 모든 API 실패 모드에 대한 에러 처리 요구사항이 정의되어 있는가? [Gap]
- [ ] CHK002 - 모든 대화형 요소에 대한 접근성 요구사항이 명시되어 있는가? [Completeness]

## 요구사항 명확성 (Clarity)
- [ ] CHK003 - '빠른 로딩'이 구체적인 시간 임계값으로 수치화되어 있는가? [Clarity, Spec §NFR-2]
- [ ] CHK004 - '관련 에피소드' 선택 기준이 명시적으로 정의되어 있는가? [Clarity, Spec §FR-5]

## 요구사항 일관성 (Consistency)
- [ ] CHK005 - 모든 페이지에서 네비게이션 요구사항이 일관되는가? [Consistency, Spec §FR-10]

## 시나리오 커버리지 (Coverage)
- [ ] CHK006 - zero-state 시나리오(데이터 없음)에 대한 요구사항이 정의되어 있는가? [Coverage, Edge Case]

## 측정 가능성 (Measurability)
- [ ] CHK007 - 시각적 계층 구조 요구사항이 측정/테스트 가능한가? [Acceptance Criteria, Spec §FR-1]
```

## 품질 차원별 체크 항목

### 완전성 (Completeness)
- 모든 필요한 요구사항이 문서화되어 있는가?

### 명확성 (Clarity)
- 요구사항이 구체적이고 모호하지 않은가?

### 일관성 (Consistency)
- 요구사항들이 서로 충돌 없이 정렬되는가?

### 측정 가능성 (Measurability)
- 성공 기준이 측정 가능한가?

### 커버리지 (Coverage)
- 모든 흐름/케이스가 다뤄졌는가?

### 엣지 케이스 (Edge Cases)
- 경계 조건이 정의되어 있는가?

## 체크리스트 항목 작성법

각 항목은 다음 패턴을 따름:
- 요구사항 품질에 대한 질문 형식
- 스펙에 작성된 (또는 작성되지 않은) 내용에 초점
- 품질 차원을 괄호 안에 표시 [Completeness/Clarity/Consistency 등]
- 기존 요구사항 확인 시 스펙 섹션 참조 [Spec §X.Y]
- 누락된 요구사항 확인 시 [Gap] 마커 사용

## 체크리스트 유형 예시

### UX 요구사항 품질: ux.md

- "모든 보호 대상 리소스에 대한 인증 요구사항이 지정되어 있는가?"
- "민감 정보에 대한 데이터 보호 요구사항이 정의되어 있는가?"

### 성능 요구사항 품질: performance.md
