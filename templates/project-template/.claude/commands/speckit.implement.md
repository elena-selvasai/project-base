# /implement 명령어

$ARGUMENTS 작업을 구현합니다.

## 개요

tasks.md의 작업을 순서대로 또는 지정된 작업을 구현합니다.

## 실행 단계

### 1. 컨텍스트 확인

다음 파일들을 읽어 구현 컨텍스트 파악:
- **필수**: tasks.md (작업 목록과 실행 계획)
- **필수**: plan.md (기술 스택, 아키텍처, 파일 구조)
- **선택**: data-model.md (엔티티와 관계)
- **선택**: contracts/ (API 명세)
- **선택**: research.md (기술 결정사항)

### 2. 프로젝트 설정 확인

기술 스택에 따라 ignore 파일 생성/확인:
- `.gitignore` (Git 저장소인 경우)
- `.dockerignore` (Docker 사용시)
- `.eslintignore` (ESLint 사용시)

**기술별 공통 패턴**:
- **Node.js**: `node_modules/`, `dist/`, `build/`, `*.log`, `.env*`
- **Python**: `__pycache__/`, `*.pyc`, `.venv/`, `venv/`, `dist/`
- **Java**: `target/`, `*.class`, `*.jar`, `.gradle/`, `build/`
- **Go**: `*.exe`, `*.test`, `vendor/`, `*.out`

### 3. 작업 실행

tasks.md의 구조에 따라 단계별 실행:

1. **단계별 실행**: 각 Phase를 순서대로 완료
2. **의존성 존중**: 순차 작업은 순서대로, [P] 작업은 병렬 가능
3. **파일 기반 조율**: 같은 파일을 수정하는 작업은 순차 실행
4. **검증 체크포인트**: 각 단계 완료 후 확인

### 4. 구현 실행 규칙

1. **Setup 먼저**: 프로젝트 구조, 의존성, 설정 초기화
2. **테스트 포함**: 필요시 테스트 코드 작성
3. **Core 개발**: 모델, 서비스, CLI 명령, 엔드포인트 구현
4. **통합 작업**: DB 연결, 미들웨어, 로깅, 외부 서비스
5. **마무리 및 검증**: 단위 테스트, 성능 최적화, 문서화

## 코딩 표준 (Quick Reference)

### 네이밍
- 파일명: `kebab-case` (예: `user-service.js`)
- 변수/함수: `camelCase` (예: `getUserById`)
- 클래스: `PascalCase` (예: `UserController`)
- 상수: `UPPER_SNAKE_CASE` (예: `MAX_RETRY_COUNT`)

### 코드 구조
- 함수는 단일 책임 원칙(SRP) 준수
- 함수는 20줄 이하로 유지
- 깊은 중첩 피하기 (최대 3단계)
- Early return 활용

### 에러 처리
- 모든 에러는 적절히 처리
- 구체적인 에러 타입 사용
- 에러 로깅 포함

### 주석
```javascript
// Good: 왜(Why)를 설명
// 캐시를 사용하여 반복적인 API 호출 방지
const cachedData = cache.get(key);

// Bad: 무엇(What)을 반복
// 변수에 값 할당
const x = 5;
```

## 진행 상태 추적

- 각 작업 완료 후 진행 상황 보고
- 비병렬 작업 실패 시 실행 중단
- 병렬 작업 [P]은 성공한 작업 계속, 실패한 작업 보고
- 명확한 에러 메시지와 디버깅 컨텍스트 제공
- **중요**: 완료된 작업은 tasks.md에서 `[X]`로 표시

## 완료 검증

- 모든 필수 작업 완료 확인
- 구현된 기능이 원래 명세와 일치하는지 확인
- 테스트 통과 및 커버리지 요구사항 충족 확인
- 구현이 기술 계획을 따르는지 확인
- 완료된 작업 요약과 함께 최종 상태 보고

## 문제 발생 시

디버깅 과정에서:
- 원인 분석이 되지 않거나
- 반복적으로 수정을 시도하는 경우

→ 작업을 즉시 중단하고 상황을 보고:
1. 현재까지 파악된 상황
2. 시도한 방법
3. 해결되지 않는 원인

**참고**: 이 명령어는 tasks.md에 완전한 작업 분류가 있다고 가정합니다. 작업이 불완전하거나 누락된 경우 먼저 `/tasks`를 실행하여 작업 목록을 재생성하세요.
