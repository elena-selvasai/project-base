# /tasks 명령어

$ARGUMENTS 경로의 PLAN.md를 세분화된 작업 체크리스트(tasks.md)로 분류해주세요.

## 지침

1. **입력**: PLAN.md 파일 경로
2. **출력**: 같은 디렉토리에 tasks.md 생성
3. **참조 파일**:
   - 해당 PLAN.md - 구현 단계 참조
   - 해당 SPEC.md - 요구사항 확인

## tasks.md 형식

```markdown
# 작업: [기능 이름]

## 1단계: [단계 이름]
- [ ] 작업 1 설명
- [ ] 작업 2 설명
- [ ] 작업 3 설명

## 2단계: [단계 이름]
- [ ] 작업 4 설명
- [ ] 작업 5 설명
...
```

## 작업 분류 원칙

### 원자성
- 각 작업은 1-2시간 내에 완료 가능해야 함
- 너무 큰 작업은 더 작게 분류
- 너무 작은 작업은 병합

### 의존성 순서
- 선행 작업이 먼저 오도록 정렬
- 의존성이 있으면 명시

### 필수 포함 항목
- 설정/환경 구성 작업
- 구현 작업
- 테스트 작업 (단위, 통합, E2E)
- 문서화 작업

### 작업 설명 작성법

✅ 좋은 예:
```
- [ ] CartService.addItem() 메서드 구현 - 재고 검증 포함
- [ ] User 모델에 lastLogin 필드 추가
- [ ] /api/cart POST 엔드포인트 생성
```

❌ 나쁜 예:
```
- [ ] 서비스 구현
- [ ] 코드 작성
- [ ] 테스트
```

## 단계 구성 예시

1. **설정 (Setup)**: 디렉토리 구조, 의존성 설치, 마이그레이션
2. **데이터 레이어 (Data Layer)**: 모델, 스키마, 리포지토리
3. **비즈니스 로직 (Business Logic)**: 서비스, 유틸리티
4. **API 레이어 (API Layer)**: 컨트롤러, 라우터, 미들웨어
5. **테스트 (Testing)**: 단위, 통합, E2E 테스트
6. **문서화 (Documentation)**: API 문서, README 업데이트

## 진행 상태 표시

작업 완료 시 `[x]`로 표시:
```markdown
- [x] 완료된 작업
- [ ] 진행 중인 작업
```
